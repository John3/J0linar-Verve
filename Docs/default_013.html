VController Methods

addChild( SimObject pObject )
Add a child object to this node.
No return value.
Required Parameters:
pObject-The SimObjectID of the object to be added to this node.
addDataField( string pFieldType, string pFieldName )
Add a new data entry to the Data Table.
No return value.
Required Parameters:
pFieldType-The method of evaluating the field's data.
pFieldName-The name of the field to be added to the Data Table.
clear( void )
Detaches and deletes all of the child objects.
No return value.
Example:
%myController.clear();
getCount( void )
Get the number of child objects.
Returns the number of child objects.
getDataFieldCount( void )
Get the number of data elements in the Data Table.
Returns the size of the Data Table.
Example:
// Fetch the Number of Data Fields.
%dataFieldCount = %myController.getDataFieldCount();
for ( %i = 0; %i < %dataFieldCount; %i++ )
{
    // Fetch the name of the ith Field.
    %dataFieldName  = %myController.getDataField( %i );

    // Fetch the Field Type.
    %dataFieldType  = %myController.getDataFieldType( %dataFieldName );

    // Fetch the evaluated Value.
    %dataFieldValue = %myController.getDataFieldValue( %dataFieldName );
}
getDataFieldName( int pIndex )
Get the name of the field given by the passed index.
Returns the name of the field corresponding to the given index.
Required Parameters:
pIndex-The index of the data field you wish to check.
Example:
// See VController::getDataFieldCount()
getDataFieldType( string pFieldName )
Get the type of data for the given field.
Returns the data type.
Required Parameters:
pFieldName-The name of the field you wish to check.
Example:
// See VController::getDataFieldCount()
getDataFieldValue( string pFieldName )
Get the evaluated data from the data field.
Returns the evaluated data from the field.
Required Parameters:
pFieldName-The name of the field you wish to evaluate.
Example:
// See VController::getDataFieldCount()
getObject( int pIndex )
Get the object corresponding to the given index.
Returns the SimObjectID for the object.
Required Parameters:
pIndex-The index of the object you wish to retrieve.
getTimeScale( void )
Get the playback speed. A value > 0.0 will enable the Controller to play forwards, while a value < 0.0 will play backwards.
Playback Speed.
isDataField( string pFieldName )
Is the field a member of the Data Table?
Returns true if the field is a member of the Data Table.
Required Parameters:
pFieldName-The name of the dynamic field you wish to check.
Example:
// Valid Data Field?
if ( %myController.isDataField( %myField ) )
{
    // ...
}
isPaused( void )
Is the sequence currently paused?
Returns true if the Controller is paused.
Example:
// Paused?
if ( %myController.isPaused() )
{
    // ...
}
isPlaying( void )
Is the sequence currently playing?
Returns true if the Controller is playing.
Example:
// Playing?
if ( %myController.isPlaying() )
{
    // ...
}
isStopped( void )
Is the sequence currently stopped?
Returns true if the Controller is stopped.
Example:
// Stopped?
if ( %myController.isStopped() )
{
    // ...
}
pause( void )
Pause the sequence. Playback can resume by calling VController::play().
No return value.
Example:
%myController.pause();
play( [int pTime] )
Play the sequence. If a value for pTime is specified, the Controller is reset and played from that time.
No return value.
Optional Parameters:
pTime-The time to start playing the sequence from.
Example:
// Play from the current position.
%myController.play();

// Play from 1 second.
%myController.play( 1000 );
readFile( string pFileName )
Clears the object and loads the new data from the given filename.
Returns true if the read was successful.
Required Parameters:
pFileName-The target file to read from.
Example:
%myController.readFile( "sequences/mySequence.vsf" );
readTemplate( string pFileName )
Load data from given filename.
Returns true if the read was successful.
Required Parameters:
pFileName-The target file to read from.
removeChild( SimObject pObject )
Remove the target object from this node.
No return value.
Required Parameters:
pObject-The SimObjectID of the object to be removed from this node.
removeDataField( string pFieldName )
Remove a data entry from the Data Table.
No return value.
Required Parameters:
pFieldName-The name of the field to be removed from the Data Table.
reset( [int pTime] )
Reset the Controller's and child object's state.
No return value.
Optional Parameters:
pTime-The target time to reset to.
Example:
%myController.reset();
setTimeScale( float pTimeScale )
Set the playback speed. A value > 0.0 will enable the Controller to play forwards, while a value < 0.0 will play backwards. If |pTimeScale| > 1.0, then playback will be faster than normal, while |pTimeScale| < 1.0 will be slower.
No return value.
Required Parameters:
pTimeScale-Playback speed.
Example:
// Play Forwards at 200% Speed.
%myController.setTimeScale( 2.0 );

// Play Backwards at 50% Speed.
%myController.setTimeScale( -0.5 );
sortGroups( void )
Sort Groups by their Labels.
No return value.
sortTracks( void )
Sort Tracks by their Labels.
No return value.
step( void )
Step forward one frame.
No return value.
Example:
%myController.step();
stop( void )
Stop the sequence.
No return value.
Example:
%myController.stop();
writeFile( string pFileName )
Save to a given filename.
Returns true if the write was successful.
Required Parameters:
pFileName-The target file to write to.

VController Fields

Duration-Total length of the sequence (in milliseconds).
Loop-Instead of stopping once playback is complete, the Controller will reset and resume play.
LoopBackwards-When the sequence loops, reverse the direction of play.
LoopCount-The number of times the sequence loops before stopping. -1 will cause the sequence to loop indefinitely.
LoopDelay-When the sequence loops, delay playback by this value (in milliseconds).
ResetOnCompletion-When the sequence is completed, reset the state of the Controller.
Time-Current position of the Controller (in milliseconds).
TimeScale-Speed of playback. A value > 0.0 will enable the Controller to play forwards, while a value < 0.0 will play backwards. If |TimeScale| > 1.0, then playback will be faster than normal, while |TimeScale| < 1.0 will be slower.

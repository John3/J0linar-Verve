VPath Methods

addNode( transform pTransform, float pWeight, [int pLocation] )
Add a node with the given properties. Nodes represent physical points that attached objects move towards or between, but the PathType determines "how" they move between them.
No return value.
Required Parameters:
pTransform-The position and rotation of the new node.
pWeight-The weight of the new node.
Optional Parameters:
pLocation-The index of the new node.
Example:
// Add node to the end of the path.
%myPath.addNode( "130.17 10.68 -156.16 1.0 0.0 0.0 1.54", 10 );

// Insert a new node into the path at index 2 (i.e. node #3).
%myPath.addNode( "130.17 10.68 -156.16 1.0 0.0 0.0 1.54", 10, 2 );
attachObject( SimObject pObject, bool pForward, float pSpeed, bool pRelative, int pStartNode, [int pEndNode] )
Attach an object to this path with the given properties. If the object is already attached to a path, then a warning will be displayed and the object will *not* be attached to this path.
No return value.
Required Parameters:
pObject-The SimObjectID of the object to be attached.
pForward-Should the object be moving forward?
pSpeed-The speed that the object will travel around the path.
pRelative-Offset the object based on the difference between the start node and its current position.
pStartNode-The index of the node this object starts pathing from.
Optional Parameters:
pEndNode-The index of the node this object will stop pathing at
Example:
// Attach an object moving forward, starting from the first node and never
// stopping.
%myPath.attachObject( %myObject, true, 15.0, false, 0 );
deleteNode( int pNodeIndex )
Delete the node with the given index. If you delete a node that an attached object is moving to, or from then the object's movement will adjust so that it has a valid path.
No return value.
Required Parameters:
pNodeIndex-The index of the node to be deleted.
Example:
// Delete the third node in the path.
%myPath.deleteNode( 2 );
detachObject( SimObject pObject )
Detach the object from this path in place.
No return value.
Required Parameters:
pObject-The SimObjectID of the object to be detached.
Example:
%myPath.detachObject( %myObject );

getPathPosition( int pSrcNodeIndex, int pDstNodeIndex, int pInterp )
Get the world position of the path at the interp point between two nodes.
Returns the world position of the interp time between the two given nodes.
Required Parameters:
pSrcNodeIndex-The first node.
pDstNodeIndex-The second node.
pInterp-The time to interp between the two nodes. Value is between 0.0 and 1.0.
Example:
// Get the position of the path 50% of the  way between the first two nodes.
%pathPosition = %myPath.getPathPosition( 0, 1, 0.5 );

setPathObjectActive( SimObject pObject, bool pActive )
Enable or disable the object from traveling around this path. Inactive objects are still attached to the path, but are not updated.
No return value.
Required Parameters:
pObject-The SimObjectID of the object being altered.
pActive-The new status of the object.
Example:
// Stop the object moving around the path, but don't detach it.
%myPath.setPathObjectActive( %myObject, false );
setPathObjectEndNode( SimObject pObject, bool pNodeIndex )
Set end node of the path object. If a value of "-1" is applied, the object will path indefinitely.
No return value.
Required Parameters:
pObject-The SimObjectID of the object being altered.
pNodeIndex-The index of the node that the object will cease pathing upon reaching.
Example:
// Stop the object moving around the path once it reaches this node.
%myPath.setPathObjectEndNode( %myObject, 2 );

setPathObjectInterp( SimObject pObject, float pInterp )
Set the interp position of the object between its current nodes.

Move the object to the node's position. You may also want to observe the "setPathObjectInterp" method.
No return value.
Required Parameters:
pObject-The SimObjectID of the object being altered.
pNodeIndex-The index of the node that the object will reposition to.
Example:

setPathObjectOffset( SimObject pObject, vector pOffset )
Set the position offset of the object. As the object is moving along the path, its position is offset by this value. Setting the "Relative" parameter while attaching an object will automatically apply an offset value.
No return value.
Required Parameters:
pObject-The SimObjectID of the object being altered.
pOffset-The new position offset of the object.
Example:
// Offset the object 5 units above the path.
%myPath.setPathObjectOffset( %myObject, "0 0 5" );
setPathObjectOrientationMode( SimObject pObject, string pOrientationType, [SimObject pObject / vector pPoint] )
Set the orientation mode of the object. This property affects the rotation of the object. If you wish to ignore the object's rotation altogether, set the mode to "FREE".
No return value.
Required Parameters:
pObject-The SimObjectID of the object being altered.
pOrientationType-The new orientation type of the object.
Optional Parameters:
pObject-If the orientation type is set to OBJECT, this parameter must be the SimObjectID of a scene object.
pPoint-If the orientation type is set to POINT, this parameter must be a vector.
Example:
//----------------------------------------------------------------------------
// List of valid orientation modes:
//   - FREE
//   - INTERPOLATE
//   - TOOBJECT
//   - TOPATH
//   - TOPOINT
//----------------------------------------------------------------------------

// Set the object to look at another object.
%myPath.setPathObjectOrientationMode( %myObject, "TOOBJECT", %myOtherObject );

// Set the object to look at a world coordinate.
%myPath.setPathObjectOrientationMode( %myObject, "TOPOINT", "0 0 20" );

setPathType( string pPathType )
The path type dictates how attached objects move between nodes. There are currently two supported path types, "BEZIER" and "LINEAR".
No return value.
Example:
//----------------------------------------------------------------------------
// List of valid path types:
//   - LINEAR
//   - BEZIER
//----------------------------------------------------------------------------

%myPath.setPathType( "BEZIER" );

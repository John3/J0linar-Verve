Extension Classes

Camera
Camera groups work together with the Director Group and Track to change camera view of the sequence.

While you can have any number of Camera Groups, there can only be one Group active at any point in time. If a Group is active, the Game's render view is set to the object that the Group is referencing. The active Group is referenced in the root Controller from the method, VController::getActiveCamera().
Objects:
  • VCameraGroup
  • VCameraShakeEvent
  • VCameraShakeTrack
  • VCameraTrack
  • VPostEffectToggleEvent
  • VPostEffectToggleTrack
Director
This group is unique - you can only have one Director Group in your sequence.

The Director Group, coupled with a Director Track, gives you power over scene definition and camera views in your sequence. This is particularly useful for Cinematic or Cutscene sequences! You wouldn't need to use a Director if you were, say moving an elevator or opening a door. In this case, you will remain in control of your Player, Vehicle or Camera.
Objects:
  • VDirectorEvent
  • VDirectorGroup
  • VDirectorTrack
  • VFadeEvent
  • VFadeTrack
  • VSceneJumpEvent
  • VSceneJumpTrack
  • VSlowMoEvent
  • VSlowMoTrack

Movable Object
All Scene Objects in Torque 3D, with the exception of TSStatic, can be used by these classes. A Motion Track gives direct control over when and how an object paths.

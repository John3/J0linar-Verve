Tutorial : Shape Object Animation

Objects: VShapeAnimationTrack, VShapeAnimationEvent

An Animation Event will tell the reference object to start playing an animation upon triggering. The animation that is to be played is defined by the Event's "AnimationData" property. If an invalid animation name is specified, no animation will play.

Animations which play as a result of a VAnimationEvent will be adjusted so that the correct position and timescale are applied. Animations which allow for transitions or loop are treated no differently.

While VAnimationTrack can be added to any VSceneObjectGroup object, the SceneObject that you intend to animate must be derived from the 'ShapeBase' class. This includes commonly used object types like Player, Vehicle and StaticShape, but please note that TSStatic does not derive from ShapeBase and cannot play animations.

Since the ShapeBase class networks animation thread properties, Playing or Pausing the root Controller will affect playback on all clients.


How to Use:

To animate an object, you must first setup a valid VSceneObjectGroup and then create a new Track. If you are unsure of how to create or validate a Group, click here.

Right-Click on the Group, go to "Add Track" and select "Add Animation Track".

Right-Click on the Track and select "Add Event".

Select the newly created Event. The Property List window will refresh and inspect the Event. Change the "AnimationData" property to the name of the animation you wish to have played upon triggering.
